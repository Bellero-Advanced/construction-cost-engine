\section{สรุปผลการวิจัย}
\label{sec:summary}

งานวิจัยนี้ได้พัฒนาระบบอัตโนมัติสำหรับการคำนวณต้นทุนต่อหน่วยของวัสดุ
ก่อสร้าง โดยรวบรวมข้อมูลจากหน่วยงานภาครัฐและสถานประกอบการค้าปลีก

100 ผ่านการบูรณาการระหว่างกลยุทธ์อัตโนมัติและกลยุทธ์ AI Agent.

\section{สรุปผลตามวัตถุประสงค์ของงานวิจัย}
\label{sec:summary-by-objective}

\subsection{การพัฒนาระบบดึงข้อมูลอัตโนมัติ}

ระบบสามารถเชื่อมต่อกับ 11 แหล่งข้อมูล ครอบคลุม 3 หน่วยงานภาครัฐ
(TPSO, CGD, DIT) และผู้ค้าปลีกสมัยใหม่ 8 ราย ผ่าน 4 กลยุทธ์การดึง
ข้อมูล ได้แก่ การถอดข้อความจากเอกสาร PDF ด้วย \texttt{unpdf}
การเรียก JSON API ผ่าน Endpoint \texttt{suggest.jsp} การใช้บริการ
Proxy ของ ScrapingBee ร่วมกับ Cloudflare Browser Rendering และ
การใช้ AI Agent Research สำหรับแหล่งที่ไม่สามารถดึงข้อมูลได้
ด้วยวิธีอัตโนมัติ.

\subsection{การพัฒนาระบบคำนวณต้นทุนวัสดุต่อหน่วย}

ระบบรองรับการคำนวณบัญชีแสดงปริมาณงาน (Bill of Quantities) ครบ 5
ประเภทงานตามหลักเกณฑ์การคำนวณราคากลางงานก่อสร้างของกรมบัญชีกลาง
(ตุลาคม พ.ศ.\ 2560) ตามที่ระบุในขอบเขตของการศึกษา (หัวข้อ
\ref{sec:scope-work-types}) ได้แก่ งานก่ออิฐ งานบุกระเบื้องผนังและพื้น
งานเสาเอ็นและคานทับหลังคอนกรีตเสริมเหล็ก งานเหล็กเสริมคอนกรีต และ
งานทาสี.

ทุกการคำนวณจะถูกบันทึกอัตโนมัติเป็นข้อมูลข้อเท็จจริงในตาราง
\texttt{fact\_calculations} บน Cloudflare KV เพื่อใช้สำหรับการ
วิเคราะห์เชิงรวม (Aggregate Analytics) ในอนาคต.

\subsection{การวิเคราะห์เชิงเวลาและพื้นที่}

ระบบให้บริการเครื่องมือวิเคราะห์เชิงสถิติ 4 ประเภท ได้แก่
\begin{itemize}[leftmargin=2em,itemsep=0.1em]
  \item \textbf{การวิเคราะห์แนวโน้มเชิงเส้น (Linear Trend Analysis)}
        ในรูป $Y = a + bX$ พร้อมค่าสัมประสิทธิ์การกำหนด $R^2$
        เพื่อใช้พยากรณ์แนวโน้มราคา
  \item \textbf{อัตราการเปลี่ยนแปลงร้อยละแบบเดือนต่อเดือน (Month-over-Month
        Percentage Change)} เพื่อวัดความผันผวนรายเดือน
  \item \textbf{การตรวจจับค่าผิดปกติ (Outlier Detection)} ด้วยวิธี
        z-score โดยใช้เกณฑ์ $|z| > 2.0$ เพื่อกรองข้อมูลที่อาจคลาดเคลื่อน
  \item \textbf{การรวบรวมข้อมูลรายภูมิภาค (Regional Aggregator)}
        คำนวณค่ามัธยฐานของราคาใน 6 ภูมิภาค เพื่อใช้ปรับค่าตัวคูณ
        เชิงพื้นที่ (Regional Adjustment Factor)
\end{itemize}

จากการทดสอบกับข้อมูลจริงที่ระบบรวบรวมได้ (สแนปช็อต ณ วันที่ 25
พฤษภาคม พ.ศ.\ 2569) พบว่าราคาปูนซีเมนต์ปอร์ตแลนด์ประเภทที่ 1 ใน
กรุงเทพมหานครมีแนวโน้มเพิ่มขึ้น 0.819 บาท/เดือน ($R^2 = 0.871$)
ตลอด 10 เดือนที่ศึกษา และราคาในภาคใต้สูงกว่าราคาในภาคกลางคิดเป็น
ร้อยละ 22.9 ซึ่งเป็นข้อมูลสำคัญสำหรับการประมาณราคาในแต่ละพื้นที่.

\textbf{สรุปตัวเลขสำคัญ:} ระบบรวบรวมข้อมูลราคาได้ครบ 297 จาก 297
รายการ (11 แหล่ง $\times$ 27 วัสดุ คิดเป็นร้อยละ 100), ราคาตลาด
สูงกว่าราคาฐานของกรมบัญชีกลางเฉลี่ยร้อยละ 8.5 (ค่ามัธยฐานร้อยละ 8.0)
จาก 37 คู่ราคาที่เปรียบเทียบได้ ราคาปูนซีเมนต์มีแนวโน้มเพิ่มขึ้น
ร้อยละ 4.9 ในช่วง 10 เดือน และราคาในภาคใต้สูงกว่าราคาในภาคกลาง
ร้อยละ 22.9.

\section{ส่วนสนับสนุนเชิงวิชาการ (Academic Contribution)}
\label{sec:contribution}

จากการทบทวนวรรณกรรมในบทที่ 2 พบว่างานวิจัยที่ผ่านมา
\cite{rao2015,chunyaem2019,liu2024,lee2024} ได้พัฒนาแนวทางเฉพาะส่วน
ของกระบวนการ เช่น Web Scraping, การจำแนกประเภทด้วยปัญญาประดิษฐ์
(AI Classification), การตรวจจับค่าผิดปกติ (Outlier Detection)
หรือการพยากรณ์ราคา (Price Forecasting) แต่ \emph{ยังขาดงานวิจัยที่
บูรณาการกระบวนการทั้งหมดเข้าด้วยกันในระบบเดียว}.

งานวิจัยนี้ \emph{เติมเต็มช่องว่างดังกล่าว} ด้วยการพัฒนาระบบสายงาน
(Pipeline) แบบจุดต้นถึงจุดปลาย (End-to-End) ที่ครอบคลุมทุกขั้นตอน
ดังนี้

\begin{enumerate}[leftmargin=2em,itemsep=0.2em]
  \item \textbf{การรวบรวมข้อมูลจากหลายแหล่ง (Multi-source Data
        Acquisition)} --- รองรับ 4 กลยุทธ์การดึงข้อมูลที่แตกต่างกัน
  \item \textbf{การจับคู่รายการวัสดุข้ามแหล่ง (Cross-source
        Canonicalization)} --- แก้ปัญหาการระบุเอกลักษณ์ของวัสดุ
        (Entity Matching) ด้วยตัวกรอง 3 ชั้น
  \item \textbf{การมาตรฐานหน่วย (Unit Standardization)} --- แปลง
        หน่วยที่ต่างกัน 12 ประเภทโดยอัตโนมัติ
  \item \textbf{การตรวจสอบความถูกต้องเชิงสถิติ (Statistical
        Validation)} --- ใช้ z-score ตรวจจับค่าผิดปกติร่วมกับการ
        วิเคราะห์เส้นแนวโน้ม (Linear Regression)
  \item \textbf{ระบบคำนวณบัญชีแสดงปริมาณงาน (BOQ Engine)} ---
        รองรับ 5 ประเภทงานตามหลักเกณฑ์ของทางราชการ
  \item \textbf{การวิเคราะห์เชิงพื้นที่และเวลา (Spatial-temporal
        Analytics)} --- เปรียบเทียบราคาเชิงพื้นที่ (6 ภูมิภาค) และ
        เชิงเวลา (10 เดือน)
\end{enumerate}

นอกจากนี้ การติดตั้งใช้งานระบบบนแพลตฟอร์มแบบ Edge (Cloudflare
Workers) ทำให้ระบบมีหน่วงเวลาเฉลี่ยน้อยกว่า 50 มิลลิวินาทีทั่วโลก
ซึ่งเป็นการแสดงให้เห็นถึงความเหมาะสมของสถาปัตยกรรมแบบ Serverless
สำหรับงานวิจัยลักษณะนี้ และเปิดทางสำหรับการเผยแพร่เป็น Public REST
API ในอนาคต.

\section{ข้อจำกัดของการศึกษา}
\label{sec:limit}

งานวิจัยนี้มีข้อจำกัดหลายประการที่ควรกล่าวถึงเพื่อให้การประเมินผล
มีความเป็นกลาง ดังนี้

\subsection{ข้อจำกัดด้านการเข้าถึงข้อมูล}

\begin{enumerate}[leftmargin=2em,itemsep=0.2em]
  \item \textbf{ระบบป้องกันบอทในเว็บไซต์ผู้ค้าปลีกสมัยใหม่}\\
        จากผู้ค้าปลีก 8 รายในกลุ่ม Modern Trade มี 5 ราย ได้แก่
        Thai Watsadu, BnB Home, Global House, SCG Home และ DoHome
        ที่ใช้ระบบ Cloudflare Bot Challenge หรือพัฒนาเว็บไซต์ในรูปแบบ
        Single Page Application ซึ่งโหลดราคาผ่านการเรียก XHR หลังการ
        Render ทำให้ไม่สามารถดึงข้อมูลด้วยวิธี Web Scraping แบบ
        ดั้งเดิมได้ ระบบจึงต้องใช้กลยุทธ์สำรองผ่าน ScrapingBee
        Residential Proxy ซึ่งมีค่าใช้จ่ายแปรผันตามจำนวนคำขอและจำกัด
        ความถี่การดึงข้อมูล.

  \item \textbf{การป้องกันการเข้าถึงอัตโนมัติของกรมบัญชีกลาง}\\
        เว็บไซต์ของกรมบัญชีกลางมีระบบป้องกันบอทระดับสูง ทำให้ไม่
        สามารถดึงข้อมูลได้โดยตรง ระบบจึงอาศัย AI Agent อ่านราคาจาก
        เอกสารราคามาตรฐานที่เผยแพร่ และบันทึกผ่าน Endpoint สำหรับ
        ผู้ดูแลระบบ.

  \item \textbf{การยกเลิก URL ของกรมการค้าภายใน}\\
        เว็บไซต์เดิมของกรมการค้าภายในที่เผยแพร่ราคารายวันไม่สามารถ
        เข้าถึงได้ในช่วงที่ดำเนินการวิจัย ทำให้ต้องใช้ข้อมูล CMI
        ของ TPSO เป็นตัวแทน.
\end{enumerate}

\subsection{ข้อจำกัดด้านความสดของข้อมูล}

ราคาประเมินของทางราชการ (CGD) มีรอบการปรับปรุงรายไตรมาส ระบบใช้อายุ
แคช (TTL) 30 วันร่วมกับงานตามตารางรายวัน (Daily Cron) ซึ่งเพียงพอ
สำหรับการศึกษาในระดับงานวิจัย แต่อาจไม่ทันต่อการเปลี่ยนแปลงของตลาด
ในช่วงที่ราคาวัสดุก่อสร้างผันผวนรุนแรง (เช่น ช่วงสงครามรัสเซีย-ยูเครน
ที่ราคาเหล็กเปลี่ยนแปลงรายสัปดาห์).

\subsection{ข้อจำกัดด้านความครอบคลุมของกลุ่มตัวอย่าง}

ระบบทดสอบจริงเฉพาะวัสดุก่อสร้าง 27 รายการ ครอบคลุม 77 จังหวัด แต่
ความครอบคลุมรายภูมิภาคยังบาง — ค่าเริ่มต้นส่วนใหญ่กำหนดที่จังหวัด
กรุงเทพมหานคร และมีการเก็บข้อมูลภูมิภาคจริงเพียง 5 ภูมิภาค (ผ่าน
ราคาตั้งต้นที่ AI Agent สืบค้นใน 5 จังหวัดอ้างอิง). การขยายให้
ครอบคลุมครบทั้ง 77 จังหวัดต้องอาศัยความร่วมมือกับ TPSO ในระดับ
รายงานภูมิภาค หรือความร่วมมือกับผู้ค้าปลีกท้องถิ่น.

\subsection{ข้อจำกัดด้านวิธีการวิเคราะห์เชิงสถิติ}

การวิเคราะห์เชิงสถิติของระบบเป็นวิธีพื้นฐาน (การวิเคราะห์เส้นแนวโน้ม
เชิงเส้น, z-score, อัตราการเปลี่ยนแปลงร้อยละ) ยังไม่ได้ใช้แบบจำลอง
พยากรณ์ราคาขั้นสูง เช่น แบบจำลอง ARIMA หรือ Prophet ซึ่งอาจให้
ความแม่นยำสูงกว่าในการพยากรณ์ราคาในอนาคต \cite{liu2024}. การประยุกต์
ใช้แบบจำลองดังกล่าวเป็นประเด็นที่งานวิจัยในอนาคตสามารถต่อยอดได้.

\section{ข้อเสนอแนะสำหรับงานวิจัยในอนาคต}
\label{sec:future-work}

จากข้อจำกัดข้างต้น สามารถเสนอแนวทางสำหรับงานวิจัยในอนาคตได้ 6 ประเด็น
หลัก ดังนี้

\begin{enumerate}[leftmargin=2em,itemsep=0.3em]
  \item \textbf{การวิเคราะห์ย้อนกลับของ API ภายใน (Reverse Engineering
        of Internal APIs)}\\
        วิเคราะห์ Endpoint ของการเรียก XHR ในเว็บไซต์ DoHome, SCG Home
        และ Global House เพื่อเข้าถึงข้อมูลผ่านส่วนต่อประสานภายใน
        โดยตรง โดยไม่ต้องอาศัยการจำลองเบราว์เซอร์ ซึ่งจะลดต้นทุนการใช้
        บริการ ScrapingBee และเพิ่มอัตราการดึงข้อมูลสำเร็จ.

  \item \textbf{แบบจำลองพยากรณ์ราคาขั้นสูง}\\
        ขยายจากการวิเคราะห์เส้นแนวโน้มเชิงเส้นไปสู่แบบจำลอง ARIMA,
        Prophet หรือโครงข่ายประสาทเทียมแบบ LSTM เพื่อพยากรณ์ราคาใน
        อนาคต โดยใช้ข้อมูลย้อนหลังที่ระบบสะสมจากงานตามตารางรายวัน
        การพยากรณ์ที่แม่นยำจะเป็นประโยชน์ต่อการวางแผนงบประมาณของ
        โครงการระยะยาว.

  \item \textbf{การชั่งน้ำหนักความเชื่อถือแบบปรับตัว (Adaptive Trust
        Weighting)}\\
        กำหนดน้ำหนักของแหล่งข้อมูลแบบปรับตัวอัตโนมัติ เช่น น้ำหนักของ
        ภาครัฐ 2 เท่าของภาคเอกชน หรือน้ำหนักของ TPSO 3 เท่าของ DIT
        เมื่อรวมเป็นดัชนีราคาผสม (Composite Price Index) การชั่งน้ำหนัก
        ดังกล่าวควรอ้างอิงจากความน่าเชื่อถือทางสถิติของแต่ละแหล่ง
        \cite{elmohr2022}.

  \item \textbf{การขยายความครอบคลุมรายจังหวัดให้ครบทั้ง 77 จังหวัด}\\
        ขยายการเก็บข้อมูลรายภูมิภาคให้ครอบคลุมครบทุกจังหวัด โดยใช้
        รายงาน CMI รายภูมิภาคของ TPSO เป็นแหล่งข้อมูลหลัก ร่วมกับการ
        ประสานความร่วมมือกับผู้ค้าปลีกท้องถิ่นที่มีฐานข้อมูลออนไลน์.

  \item \textbf{การเปิด Public API และการพัฒนาผู้ช่วยอัจฉริยะ}\\
        เปิด \texttt{/api-docs} ให้ผู้ประมาณราคาภายนอกเรียกใช้ได้
        พร้อมนโยบายการจำกัดอัตราการเรียกใช้ที่เหมาะสม รวมทั้งพัฒนา
        ผู้ช่วยอัจฉริยะที่ตอบคำถามในรูปแบบภาษาธรรมชาติ เช่น
        ``ราคาปูนซีเมนต์ตราเสือขนาด 50 กิโลกรัมที่จังหวัดเชียงใหม่
        ในปัจจุบันเป็นเท่าใด'' โดยใช้แบบจำลองภาษาขนาดใหญ่ร่วมกับ
        เทคนิค Retrieval-Augmented Generation (RAG) จากฐานข้อมูล KV.

  \item \textbf{การทดสอบประสิทธิภาพกับโครงการก่อสร้างจริง}\\
        เมื่อระบบรวบรวมข้อมูลราคาจากแหล่งค้าปลีกสมัยใหม่ได้ครบถ้วน
        ควรทำการทดสอบเปรียบเทียบเวลาและความแม่นยำของการจัดทำบัญชี
        แสดงปริมาณงานระหว่างวิธีดั้งเดิมกับการใช้ระบบที่พัฒนาขึ้น
        โดยใช้โครงการก่อสร้างจริงเป็นกรณีศึกษา เพื่อวัดประโยชน์เชิง
        ปฏิบัติอย่างเป็นรูปธรรม.
\end{enumerate}

\section{บทสรุปสุดท้าย}
\label{sec:final}

ระบบที่พัฒนาขึ้นในงานวิจัยนี้เป็น \emph{ต้นแบบเชิงพิสูจน์แนวคิด
(Proof of Concept)} ที่แสดงให้เห็นว่าการรวมศูนย์ข้อมูลราคาวัสดุก่อสร้าง
จากหลายแหล่งให้กลายเป็น \emph{แหล่งข้อมูลอ้างอิงเดียว (Single Source
of Truth)} สำหรับการประมาณการบัญชีแสดงปริมาณงานในประเทศไทยนั้นเป็นไป
ได้ในทางปฏิบัติ. การติดตั้งใช้งานระบบบนแพลตฟอร์มแบบ Edge (Cloudflare
Workers) ทำให้ระบบมีต้นทุนการดำเนินงานต่ำ (Scale-to-zero) และมีหน่วงเวลา
น้อยกว่า 50 มิลลิวินาทีทั่วโลก ซึ่งเหมาะสมกับการใช้งานทั้งในงานก่อสร้าง
ภาครัฐและภาคเอกชน.

ผลที่ได้จากงานวิจัยมีศักยภาพในการนำไปใช้สนับสนุน:
\begin{itemize}[leftmargin=2em,itemsep=0.1em]
  \item การประมาณราคาโครงการก่อสร้างของหน่วยงานภาครัฐที่ต้องอ้างอิง
        ราคามาตรฐานควบคู่กับราคาตลาดจริง
  \item การวิเคราะห์ความผันผวนของราคาวัสดุเพื่อใช้ในการวางแผน
        งบประมาณระยะยาว
  \item การสร้างความโปร่งใสในกระบวนการจัดซื้อจัดจ้างภาครัฐ ผ่านการ
        เปิดเผยข้อมูลราคาจากหลายแหล่งให้ผู้มีส่วนได้ส่วนเสียสามารถ
        ตรวจสอบเปรียบเทียบ
  \item การเป็นต้นแบบสำหรับงานวิจัยอื่น ๆ ที่ต้องการรวมศูนย์ข้อมูล
        ราคาสินค้าจากหลายแหล่งในบริบทของประเทศไทย
\end{itemize}

ระบบนี้แสดงให้เห็นว่า \emph{เทคโนโลยี Edge Computing, Serverless
Architecture และการบูรณาการข้อมูล (Data Integration)} สามารถนำมา
ประยุกต์ใช้ในงานวิศวกรรมโยธาเพื่อแก้ปัญหาในทางปฏิบัติของผู้ประมาณ
ราคาได้อย่างมีประสิทธิภาพ และคาดว่าผลลัพธ์ของงานวิจัยจะเป็นพื้นฐาน
สำคัญในการพัฒนาระบบสารสนเทศด้านการประมาณราคางานก่อสร้างของประเทศไทย
ในระยะยาว.
