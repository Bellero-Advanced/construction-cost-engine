Why this exists

ราคาวัสดุไทยกระจาย — ราชการ (CGD/TPSO/DIT) + ค้าปลีก (HomePro/SCG/Dohome ฯลฯ). ต้องรวมเพื่อทำ BoQ ที่อ้างอิงได้

Why Cloudflare Workers

Edge SSR + KV cache + Browser Rendering + Cron triggers ในที่เดียว. ไม่มี cold start, scale-to-zero, ราคาน้อย

Why TTL = 30 days

ราคาประเมินทางการเปลี่ยน รายไตรมาส. ดึงบ่อยกว่านี้ = เปลือง credit + ไม่ได้ value

PIPELINE / 04 · ACQUISITION

Four Strategies

A · PDF

unpdf parse

TPSO + CGD ส่งราคาใน PDF rate report. ใช้ unpdf + regex หาแถวที่ตรง material code

Input: PDF URL
Output: number

B · JSON API

suggest.jsp

HomePro/MegaHome expose /service/search/suggest.jsp?q=… JSON. ตรงเร็ว ไม่ต้อง render

Input: search term
Output: {item_name, price_display}[]

C · PROXY

ScrapingBee

Residential proxy + JS render. เก็บ JSON-LD Product → name+price แล้วกรองด้วย canonical

Input: URL + waitMs
Output: HTML → ld+json

D · MANUAL

Admin upload

POST /api/admin/upload-prices รับ CSV/JSON ใส่ KV โดยตรง. ใช้กับ govt/SPA ที่ scrape ไม่ได้

Input: {source, province, prices}
Output: KV write

PIPELINE / 05 · NORMALIZE

Option B — Canonical Spec Matching

ปัญหา: HomePro ค้นด้วยคำ "ปูน" ได้ผลลัพธ์ 200 รายการ ราคา 80-2,400฿. แต่ละแหล่งใช้คำต่างกัน ขนาดต่างกัน. ถ้า median ทั้งหมด = เปรียบเทียบไม่ตรง

1. Canonical spec per material

CEMENT_001: {
  canonical: {
    brand: "ตราเสือ",
    size:  "50kg",
    grade: "Type I"
  },
  searchTerms: [
    "ปูนซีเมนต์ปอร์ตแลนด์ ตราเสือ 50 กก",
    "ปูนตราเสือ 50 กก"
  ]
}

2. Three-tier filter

  • tight ผลลัพธ์ที่ ขนาดตรง (must) + brand/grade hit
  • loose ขนาดตรงอย่างเดียว
  • legacy ไม่มี hit เลย → median ทั้งหมด (backwards compat)

ใช้ใน _suggestJsp.ts (HomePro/MegaHome) และ _scrapingbee.ts JSON-LD path

3. UI honesty

Canonical chip
แสดง brand/size/grade ที่ใช้ filter
Trust label
Government = Official BoQ · Modern Trade = Market
Spread warning
เมื่อ spread > 30% → "Indicative range"
PIPELINE / 06 · STORAGE

Cloudflare KV — Cache + Time-Series

Live price cache

key: "{source}:{materialId}:{province}"
val: { price: 175, fetchedAt: 1716... }
TTL: 30 days (govt) / 14 days (HomePro)

อ่านที่ /api/prices/[source]/[material] ทุก request → KV hit เป็นมาตรฐาน

Weekly history snapshot

key: "hist:{source}:{material}:{prov}:{yyyy-mm-dd}"
val: { price, fetchedAt }
TTL: 365 days

เขียนทุกจันทร์ 03:17 UTC โดย GH Actions cron → ใช้สร้าง trend chart

PIPELINE / 07 · API LAYER

Endpoints

Public REST

  • GET /api/prices/[source]/[material]?province=N
    → {price, live, available, fetchedAt}
  • GET /api/compare/[material]?province=N
    → fan-out ทุก 10 sources + summary {min,max,spreadPct}
  • GET /api/sources/health?province=N
    → per-source {fresh,ok,stale,missing} + coverage%
  • GET /api/prices/status
    → live providers + ScrapingBee enabled?
  • GET /api/trend/[source]/[material]
    → time-series จาก hist: keys

Admin (x-admin-token)

  • POST /api/admin/refresh-prices?source=X
    → trigger live fetch + KV write
  • POST /api/admin/upload-prices
    → bulk ingest {source, province, prices{}}
  • POST /api/admin/snapshot-history
    → write hist: keys
  • GET /api/admin/scrapingbee-debug?source=X&material=Y
    → surface upstream status, html signals, probes
Cache headers: public, s-maxage=60, stale-while-revalidate=120 — Cloudflare edge cache ตอบ 99% ของ traffic โดยไม่แตะ Worker
